# 장애 대응 크론잡 가이드

VERIFYING 타임아웃 등 **장애/지연 건**을 주기적으로 정리하기 위한 크론 설정 방법입니다.

---

## 0. CRON_SECRET 설정 방법 (처음부터)

크론에서 API를 호출할 때 쓰는 비밀값입니다. **서버**와 **크론이 실행되는 곳** 두 군데에 같은 값을 넣으면 됩니다.

### Step 1. 시크릿 값 만들기

터미널에서 아래 중 하나만 실행해서 나온 값을 복사합니다.

```bash
# 방법 1: openssl (맥/리눅스 보통 설치됨)
openssl rand -hex 24
```

또는

```bash
# 방법 2: Python
python3 -c "import secrets; print(secrets.token_hex(24))"
```

예시로 `a1b2c3d4e5f6...` 처럼 긴 문자열이 나오면 그걸 **CRON_SECRET** 값으로 쓰면 됩니다. 한 번 만들어 두고 안전한 곳에 메모해 두세요.

---

### Step 2. API 서버에 CRON_SECRET 넣기

**API가 돌아가는 서버**(예: Coolify, Docker, PM2 등)의 **환경 변수**에 넣습니다.

| 환경 | 설정 위치 예시 |
|------|----------------|
| **Coolify** | 해당 서비스 → Environment Variables → 변수 이름 `CRON_SECRET`, 값에 Step 1에서 만든 문자열 입력 → 저장 후 재배포(또는 재시작) |
| **Docker / docker-compose** | `environment:` 에 `CRON_SECRET=여기에_Step1_값` 추가 후 컨테이너 재시작 |
| **로컬 .env** | 프로젝트 루트 `.env` 파일에 한 줄 추가: `CRON_SECRET=여기에_Step1_값` |

서버를 재시작(또는 재배포)해야 환경 변수가 적용됩니다.

---

### Step 3. 크론에서 같은 값 쓰기

크론이 실행되는 **같은 서버** 또는 **다른 서버**에서, Step 1에서 만든 **같은 값**을 씁니다.

- **스크립트로 한 번만 테스트할 때**
  ```bash
  export CRON_SECRET="Step1에서_복사한_값_그대로"
  export API_BASE_URL="https://api.nanum.online"
  bash PROJECT/scripts/cron_verifying_timeout.sh
  ```
- **crontab에 넣을 때**  
  아래 "4. crontab 등록"에서 `CRON_SECRET=여기_같은_값` 자리에 Step 1 값을 넣으면 됩니다.

**요약:** Step 1에서 만든 값 하나를 → API 서버 환경 변수 `CRON_SECRET` + 크론 실행 시 사용하는 `CRON_SECRET` 에 **똑같이** 넣기.

---

## 1. 수행 내용

| 잡 | 설명 | API |
|----|------|-----|
| **VERIFYING 타임아웃 처리** | `VERIFYING` / `PENDING_VERIFICATION` 상태로 `verifying_timeout_minutes` 초과한 건을 UNFIT 또는 ERROR로 변경 후 FE 콜백 URL로 전송 | `POST /api/v1/admin/jobs/cron/verifying-timeout` (크론 전용) 또는 `POST /api/v1/admin/jobs/process-verifying-timeout` (관리자 인증) |

판정 규칙(`judgment_rule_config`)에서 `verifying_timeout_minutes`가 0보다 클 때만 동작합니다. 0이면 비활성입니다.

---

## 2. 인증 방식

### 방법 A: 크론 전용 시크릿 (권장)

- 서버 환경 변수에 **`CRON_SECRET`** 설정.
- 크론에서 **`X-Cron-Secret`** 헤더로 동일한 값 전달.
- 관리자 키(ADMIN_API_KEY)를 크론 서버에 둘 필요 없음.

### 방법 B: 관리자 키

- **`X-Admin-Key`** 에 `ADMIN_API_KEY` 값 전달.
- 기존 관리자 API와 동일한 인증.

---

## 3. 스크립트 실행

프로젝트 루트 기준:

```bash
# 방법 A: CRON_SECRET
export CRON_SECRET="your_cron_secret_here"
export API_BASE_URL="https://api.nanum.online"
bash PROJECT/scripts/cron_verifying_timeout.sh

# 방법 B: ADMIN_API_KEY
export ADMIN_API_KEY="your_admin_key"
export API_BASE_URL="https://api.nanum.online"
bash PROJECT/scripts/cron_verifying_timeout.sh
```

`API_BASE_URL` 미설정 시 `http://localhost:8000` 사용.

---

## 4. crontab 등록 (맥/리눅스)

```bash
crontab -e
```

예: **10분마다** 실행 (실서비스 API 주소·시크릿은 환경에 맞게 변경).

```cron
# GEMS OCR: VERIFYING 타임아웃 처리 (10분마다)
*/10 * * * * CRON_SECRET=your_cron_secret API_BASE_URL=https://api.nanum.online /Users/june/Documents/GitHub/GEMS_OCR/PROJECT/scripts/cron_verifying_timeout.sh >> /var/log/gems_cron.log 2>&1
```

- 로그 경로(`/var/log/gems_cron.log`)는 쓰기 권한 있는 경로로 변경.
- 시크릿은 `crontab -e`에 직접 넣지 말고, 스크립트에서 `source /path/to/.env.cron` 등으로 읽게 하거나, 시스템 환경 변수로 두는 편이 안전합니다.

---

## 5. 서버 측 설정

1. **환경 변수**
   - 크론 전용 호출을 쓸 경우: `CRON_SECRET` 설정 (예: 32자 랜덤 문자열).
   - 판정 규칙에서 VERIFYING 타임아웃 사용 시: 관리자 페이지 또는 API로 `verifying_timeout_minutes`(분), `verifying_timeout_action`(UNFIT | ERROR) 설정.

2. **크론 호출 대상**
   - 같은 서버에서 크론이 돌면 `API_BASE_URL=http://127.0.0.1:8000` 또는 내부 URL.
   - 별도 크론 서버에서 호출하면 `API_BASE_URL=https://api.nanum.online` 등 외부 URL + `CRON_SECRET` 사용.
