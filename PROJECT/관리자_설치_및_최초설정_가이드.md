## 1단계: DB 마이그레이션 (DBeaver) — 이미 실행하셨다면

DBeaver에서 FK 오류가 났다면, **한 번에 전체가 아니라 블록 단위로** 실행해 보세요.

1. **파일 사용**: `PROJECT/migrations/admin_organizations_and_users_step_by_step.sql`  
   - 위에서부터 **[1단계]** → **[2단계]** → **[3단계]** → **[4단계]** 순서로  
   - 각 블록만 선택한 뒤 **Ctrl+Enter**(또는 실행 버튼)로 실행.
2. **같은 DB 사용**: `campaigns` 테이블이 있는 **같은 데이터베이스**(예: gems)에 연결된 상태에서 실행해야 합니다.

테이블이 이미 있다면 `CREATE TABLE IF NOT EXISTS` 때문에 에러는 나지 않고, 4단계만 `column "organization_id" already exists` 같은 메시지가 나올 수 있습니다. 그때는 4단계는 건너뛰면 됩니다.

## 2단계: 패키지 설치 (PyJWT, passlib)

추가된 패키지: `PyJWT`, `passlib[bcrypt]`

### 쿨리파이에서

- **이미지 재빌드**를 하면 `requirements.txt`를 읽어서 자동 설치됩니다.
- **재배포** 시 새 이미지로 빌드되므로, **별도 터미널 명령 없이** “저장소 반영 후 재빌드·재배포”만 하면 됩니다.

### 로컬 터미널에서

프로젝트 루트에서 한 번만 실행:

```bash
cd /Users/june/Documents/GitHub/GEMS_OCR
pip install -r requirements.txt
```

또는 가상환경 사용 중이면:

```bash
pip install PyJWT "passlib[bcrypt]"
```

## 3단계: 환경 변수 (JWT_SECRET)

담당자 **로그인(이메일/비밀번호)** 을 쓰려면 `JWT_SECRET`이 있어야 합니다.

### 쿨리파이에서

1. 해당 **서비스(애플리케이션)** 선택 → **Environment Variables** (또는 설정 → 환경 변수).
2. 변수 추가:
   - **Name**: `JWT_SECRET`
   - **Value**: 32자 이상 권장, 예: `openssl rand -hex 32` 로 생성한 값.
3. 저장 후 **재시작**(Redeploy/Restart).

### 로컬에서

프로젝트 루트의 `.env` 파일에 한 줄 추가:

```env
JWT_SECRET=여기에_32자_이상_긴_비밀문자열
```

서버(uvicorn) 다시 실행.

## 4단계: 최초 슈퍼관리자 계정 만들기

**한 번만** 하면 됩니다.

### 방법 A: API로 생성 (쿨리파이·로컬 공통)

서버가 떠 있는 상태에서, **X-Admin-Key**로 담당자 생성 API를 호출합니다.

1. **ADMIN_API_KEY** 확인  
   - 쿨리파이: 해당 앱 환경 변수에 설정된 `ADMIN_API_KEY`  
   - 로컬: `.env`의 `ADMIN_API_KEY`

2. 아래처럼 호출 (이메일·비밀번호는 원하는 값으로 변경).

**로컬 터미널 예시:**

```bash
curl -X POST "http://localhost:8000/api/v1/admin/users" \
  -H "Content-Type: application/json" \
  -H "X-Admin-Key: 여기에_ADMIN_API_KEY_값" \
  -d '{
    "email": "super@example.com",
    "password": "Abc1!xyz",
    "role": "SUPER_ADMIN",
    "campaignIds": []
  }'
```

**쿨리파이(실서버) 예시:**

```bash
curl -X POST "https://api.nanum.online/api/v1/admin/users" \
  -H "Content-Type: application/json" \
  -H "X-Admin-Key: 여기에_실서버_ADMIN_API_KEY_값" \
  -d '{
    "email": "super@example.com",
    "password": "Abc1!xyz",
    "role": "SUPER_ADMIN",
    "campaignIds": []
  }'
```

- 비밀번호는 **영문 대·소문자 각 1자 이상, 숫자, 특수문자, 8자 이상**이어야 합니다.
- 성공하면 응답에 생성된 사용자(id, email, role 등)가 나옵니다. 이 계정으로 `POST /api/v1/auth/login` 후 Bearer 토큰을 쓰면 됩니다.

### 방법 B: 로컬 터미널에서 스크립트로 생성

**로컬에서만** 사용 가능(DB 접속이 되는 환경).

1. `.env`에 다음 설정:
   - `DATABASE_URL` (DB 접속 문자열)
   - `SUPER_ADMIN_EMAIL=admin@example.com`
   - `SUPER_ADMIN_PASSWORD=Abc1!xyz` (정책 만족하는 비밀번호)

2. 프로젝트 루트에서 실행:

```bash
cd /Users/june/Documents/GitHub/GEMS_OCR
SUPER_ADMIN_EMAIL=admin@example.com SUPER_ADMIN_PASSWORD='Abc1!xyz' python PROJECT/scripts/create_super_admin.py
```

- `admin_users` 테이블이 **비어 있을 때**만 생성됩니다. 이미 슈퍼관리자가 있으면 “already exists” 메시지가 나옵니다.

## 정리 표

| 단계 | 쿨리파이 | 로컬 터미널 |
|------|----------|-------------|
| **2. 패키지** | 저장소 반영 후 **재빌드·재배포** (자동) | `pip install -r requirements.txt` |
| **3. JWT** | 앱 **환경 변수**에 `JWT_SECRET` 추가 후 **재시작** | `.env`에 `JWT_SECRET` 추가 후 서버 재실행 |
| **4. 최초 슈퍼** | **방법 A**: curl로 `X-Admin-Key` + `POST /api/v1/admin/users` (role=SUPER_ADMIN) | **방법 A** 동일, 또는 **방법 B** 스크립트 실행 |

쿨리파이에서는 DB에 직접 접속하는 스크립트 실행이 어려우므로, **4단계는 방법 A(API 호출)** 로 하는 것이 가장 쉽습니다.
