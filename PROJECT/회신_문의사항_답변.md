# 회신: Complete API documents/data 문의에 대한 답변

안녕하세요.

영수증 Complete API의 **documents**와 **data** 필드 관련 문의에 대해 아래와 같이 답변드립니다.

---

## 1. documents 방식으로만 전송하는 경우, data 값을 별도로 전달하지 않아도 되는지요?

**네. documents 방식으로만 전송하시면 data 값은 별도로 전달하지 않으셔도 됩니다.**

- v1 연동은 **documents-only**로 운영하며, FE 신규 구현에서는 **data를 사용하지 않는 것**을 권장합니다.
- `receiptId`, `userUuid`, `type`, `documents`만 보내시면 되고, **data 필드는 생략**하시면 됩니다.

---

## 2. 만약 data 값이 필요하다면, 영수증 이미지가 여러 장일 경우 data를 배열 형태로 전송해야 하는지요?

**documents 방식을 사용하시는 경우 data는 사용하지 않으시면 되며, 영수증 이미지가 여러 장이어도 “여러 장” 정보는 data가 아니라 documents 배열로만 전달하시면 됩니다.**

- **여러 장의 이미지**는 `documents` 배열로 전달합니다.  
  - 예: `documents: [ { "imageKey": "receipts/uuid_a.jpg", "docType": "RECEIPT" }, { "imageKey": "receipts/uuid_b.jpg", "docType": "RECEIPT" } ]`
- `data`는 레거시(수기 보정값 등) 용도이며, 이미지 “장 수”나 “여러 장” 표현을 위한 배열 구조가 아닙니다.
- 따라서 **영수증이 여러 장이어도 data를 배열로 보내는 방식은 사용하지 않으시고, documents 배열만으로 이미지 목록을 전달**하시면 됩니다.

---

## 3. 기존처럼 documents 방식으로 전송할 때 data 값이 null이어도 문제가 없는지요?

**네. documents 방식으로 전송하실 때 data 값을 null로 두거나, 아예 필드를 생략하셔도 문제 없습니다.**

- documents-only 연동에서는 **data는 사용하지 않는 것이 권장**되므로,  
  - `data`를 **보내지 않거나(생략)**  
  - `data: null` 로 보내시는 것  
  모두 허용됩니다.
- 서버는 `documents`만으로 이미지 목록을 처리하므로, data가 없거나 null이어도 정상 동작합니다.

---

## 요약

| 문의 내용 | 답변 |
|-----------|------|
| documents만 보낼 때 data 전달 필요 여부 | **불필요** (생략 가능) |
| 여러 장일 때 data를 배열로 보낼지 여부 | **아니요.** 여러 장은 **documents 배열**로만 전달하고, data는 사용하지 않음 |
| documents 사용 시 data가 null이어도 되는지 | **가능** (null 또는 생략 모두 문제 없음) |

추가로 궁금하신 점 있으시면 말씀해 주세요.
