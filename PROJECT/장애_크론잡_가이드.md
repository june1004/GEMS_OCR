# 장애 대응 크론잡 가이드

VERIFYING 타임아웃 등 **장애/지연 건**을 주기적으로 정리하기 위한 크론 설정 방법입니다.

---

## 0. CRON_SECRET 설정 방법 (처음부터)

크론에서 API를 호출할 때 쓰는 비밀값입니다. **서버**와 **크론이 실행되는 곳** 두 군데에 같은 값을 넣으면 됩니다.

### Step 1. 시크릿 값 만들기

터미널에서 아래 중 하나만 실행해서 나온 값을 복사합니다.

```bash
# 방법 1: openssl (맥/리눅스 보통 설치됨)
openssl rand -hex 24
```

또는

```bash
# 방법 2: Python
python3 -c "import secrets; print(secrets.token_hex(24))"
```

예시로 `a1b2c3d4e5f6...` 처럼 긴 문자열이 나오면 그걸 **CRON_SECRET** 값으로 쓰면 됩니다. 한 번 만들어 두고 안전한 곳에 메모해 두세요.

---

### Step 2. API 서버에 CRON_SECRET 넣기

**API가 돌아가는 서버**(예: Coolify, Docker, PM2 등)의 **환경 변수**에 넣습니다.

| 환경 | 설정 위치 예시 |
|------|----------------|
| **Coolify** | 해당 서비스 → Environment Variables → 변수 이름 `CRON_SECRET`, 값에 Step 1에서 만든 문자열 입력 → 저장 후 재배포(또는 재시작) |
| **Docker / docker-compose** | `environment:` 에 `CRON_SECRET=여기에_Step1_값` 추가 후 컨테이너 재시작 |
| **로컬 .env** | 프로젝트 루트 `.env` 파일에 한 줄 추가: `CRON_SECRET=여기에_Step1_값` |

서버를 재시작(또는 재배포)해야 환경 변수가 적용됩니다.

---

### Step 3. 크론에서 같은 값 쓰기

크론이 실행되는 **같은 서버** 또는 **다른 서버**에서, Step 1에서 만든 **같은 값**을 씁니다.

#### Coolify에서 크론으로 같은 값 쓰기

Coolify에서 API(GEMS_OCR)를 배포한 뒤, **같은 CRON_SECRET**을 쓰는 방법은 두 가지입니다.

**방법 1) Coolify Cron Job 리소스 사용 (권장)**

1. Coolify 대시보드 → **+ Add Resource** → **Cron Job** (또는 **Scheduled Task** 등) 선택.
2. **이미지/실행 방식**
   - 예: `curlimages/curl` 같은 이미지로 매번 컨테이너가 실행되게 하거나,
   - 또는 **실행 명령**에 `curl` 호출만 넣는 형태로 설정.
3. **환경 변수**에 Step 1에서 만든 값 넣기:
   - `CRON_SECRET` = Step 1에서 복사한 값 (API 서버에 넣은 것과 **동일**)
   - `API_BASE_URL` = 배포한 API 주소 (예: `https://api.nanum.online`)
4. **실행 스케줄** 설정 (예: 10분마다 `*/10 * * * *`).
5. **실행 명령** 예시 (컨테이너 안에서 curl 사용하는 경우):
   ```bash
   curl -s -X POST \
     -H "X-Cron-Secret: $CRON_SECRET" \
     -H "Content-Type: application/json" \
     "$API_BASE_URL/api/v1/admin/jobs/cron/verifying-timeout"
   ```
   환경 변수 `CRON_SECRET`, `API_BASE_URL`이 위 3번에서 설정한 값으로 들어가면, **크론 쪽에서도 같은 값**을 쓰는 것이 됩니다.

**방법 2) Coolify 호스트 서버의 crontab 사용**

1. Coolify가 설치된 **같은 서버**에 SSH로 접속.
2. `crontab -e` 로 크론 편집.
3. 아래 한 줄 추가 (값만 Step 1·API 주소에 맞게 수정).
   ```cron
   */10 * * * * CRON_SECRET=Step1에서_복사한_값_그대로 API_BASE_URL=https://api.nanum.online /path/to/GEMS_OCR/PROJECT/scripts/cron_verifying_timeout.sh >> /var/log/gems_cron.log 2>&1
   ```
   여기서 `CRON_SECRET=` 뒤에 넣는 값이 Step 1에서 만든 값이고, API 서버(Coolify 환경 변수)에 넣은 **CRON_SECRET과 동일**해야 합니다.

**정리 (Coolify 기준)**  
- **API 서비스**: Coolify 해당 서비스 → Environment Variables → `CRON_SECRET` = Step 1 값  
- **크론**: Coolify Cron Job의 환경 변수 또는 호스트 crontab에서 `CRON_SECRET` = **같은 Step 1 값**  
이렇게 두 곳에 **같은 값**을 넣으면 Step 3 완료입니다.

---

#### Coolify "New Scheduled Task" 폼에 넣을 값

Coolify에서 **Scheduled Task** 모달(New Scheduled Task)을 띄웠을 때, 아래처럼 넣으면 됩니다.

| 필드 | 입력 값 |
|------|----------|
| **Name** | `GEMS OCR VERIFYING 타임아웃` (또는 원하는 이름) |
| **Command** | 아래 "Command 예시" 중 하나 그대로 복사 |
| **Frequency** | `*/10 * * * *` (10분마다. Presigned 10분과 맞춰 두기 좋음. 부담 시 `*/15 * * * *` 15분) |
| **Timeout (seconds)** | `60` (1분이면 충분) |
| **Container name** | GEMS_OCR 앱의 **메인 컨테이너 이름**을 **정확히** 입력. 예: `gems-ocr`, `gems_ocr-1` 등. "GEMS"처럼 짧게만 넣으면 **다른 컨테이너를 가리키거나 없어서** exit code 3 등 오류가 납니다. Coolify 해당 서비스 → 컨테이너 목록에서 표시되는 이름을 복사해 넣으세요. |

**Command 예시 (curl 있는 컨테이너일 때)**  
컨테이너에 `curl`이 있으면:
```bash
curl -s -X POST -H "X-Cron-Secret: $CRON_SECRET" -H "Content-Type: application/json" "$API_BASE_URL/api/v1/admin/jobs/cron/verifying-timeout"
```

**Command 예시 (curl 없고 Python만 있을 때)**  
Python만 있는 컨테이너면:
```bash
python3 -c "
import os, urllib.request
url = (os.environ.get('API_BASE_URL') or 'http://localhost:8000').rstrip('/') + '/api/v1/admin/jobs/cron/verifying-timeout'
req = urllib.request.Request(url, method='POST', headers={'X-Cron-Secret': os.environ.get('CRON_SECRET',''), 'Content-Type': 'application/json'})
urllib.request.urlopen(req, timeout=60)
"
```

**필수:** 이 Scheduled Task가 도는 **컨테이너**에 환경 변수 `CRON_SECRET`, `API_BASE_URL` 이 들어가 있어야 합니다.  
→ 해당 GEMS_OCR 서비스 → **Environment Variables** 에 `CRON_SECRET`(Step 1 값), `API_BASE_URL`(예: `https://api.nanum.online`) 추가 후 저장·재배포.

---

- **스크립트로 한 번만 테스트할 때**
  ```bash
  export CRON_SECRET="Step1에서_복사한_값_그대로"
  export API_BASE_URL="https://api.nanum.online"
  bash PROJECT/scripts/cron_verifying_timeout.sh
  ```
- **일반 서버 crontab에 넣을 때**  
  아래 "4. crontab 등록"에서 `CRON_SECRET=여기_같은_값` 자리에 Step 1 값을 넣으면 됩니다.

**요약:** Step 1에서 만든 값 하나를 → API 서버 환경 변수 `CRON_SECRET` + 크론 실행 시 사용하는 `CRON_SECRET` 에 **똑같이** 넣기.

## 1. 수행 내용

| 잡 | 설명 | API |
|----|------|-----|
| **VERIFYING 타임아웃 처리** | `VERIFYING` / `PENDING_VERIFICATION` 상태로 `verifying_timeout_minutes` 초과한 건을 UNFIT 또는 ERROR로 변경 후 FE 콜백 URL로 전송 | `POST /api/v1/admin/jobs/cron/verifying-timeout` (크론 전용) 또는 `POST /api/v1/admin/jobs/process-verifying-timeout` (관리자 인증) |

판정 규칙(`judgment_rule_config`)에서 `verifying_timeout_minutes`가 0보다 클 때만 동작합니다. 0이면 비활성입니다.

---

### Presigned URL 10분과 크론 주기

| 항목 | 의미 |
|------|------|
| **Presigned URL 10분** | 업로드용 URL 유효 시간(기본 600초). 사용자가 URL 발급 후 **10분 안에** 업로드를 완료해야 함. |
| **크론 주기** | VERIFYING/PENDING_VERIFICATION 상태로 **지나치게 오래 대기한 건**을 몇 분마다 점검해 정리할지. |

둘은 **목적이 다릅니다.** Presigned 10분에 맞춰 크론을 꼭 10분으로 할 필요는 없지만, 같은 10분으로 두면 기억하기 쉽고, 지연 건도 10분마다 빠르게 정리됩니다.

**권장**

| 주기 | cron 표현식 | 설명 |
|------|-------------|------|
| **10분 (권장)** | `*/10 * * * *` | Presigned 10분과 맞춰 두기 좋고, 지연 건을 10분마다 처리. |
| **15분** | `*/15 * * * *` | 호출 횟수 조금 줄이고 싶을 때. `verifying_timeout_minutes`가 30분 이상이면 15분으로 해도 충분. |

**결론:** Presigned가 10분이면 크론도 **10분(`*/10 * * * *`)** 으로 두는 것을 권장. 부담되면 15분으로 설정해도 됩니다.

## 2. 인증 방식

### 방법 A: 크론 전용 시크릿 (권장)

- 서버 환경 변수에 **`CRON_SECRET`** 설정.
- 크론에서 **`X-Cron-Secret`** 헤더로 동일한 값 전달.
- 관리자 키(ADMIN_API_KEY)를 크론 서버에 둘 필요 없음.

### 방법 B: 관리자 키

- **`X-Admin-Key`** 에 `ADMIN_API_KEY` 값 전달.
- 기존 관리자 API와 동일한 인증.

## 5. 서버 측 설정

1. **환경 변수**
   - 크론 전용 호출을 쓸 경우: `CRON_SECRET` 설정 (예: 32자 랜덤 문자열).

2. **크론 호출 대상**
   - 같은 서버에서 크론이 돌면 `API_BASE_URL=http://127.0.0.1:8000` 또는 내부 URL.
   - 별도 크론 서버에서 호출하면 `API_BASE_URL=https://api.nanum.online` 등 외부 URL + `CRON_SECRET` 사용.

## 5.5 Coolify 크론: "SSH command failed with exit code 3" 해결

**증상:** Coolify Scheduled Task 로그에 `Job permanently failed after 1 attempts: SSH command failed with exit code: 3` 이 나올 때.

**원인:** `exit code 3` 은 대부분 **curl** 이 반환하는 값입니다. curl 3 = **URL malformed**(URL 형식 오류). 즉, 크론이 실행되는 환경에서 **`API_BASE_URL` 이 비어 있거나 잘못 전달**되어 URL이 깨진 경우입니다.

**해결 순서:**

1. **Coolify Scheduled Task 실행 방식 확인**
   - **Run in container** (컨테이너 안에서 실행): GEMS_OCR 서비스의 **같은 컨테이너**를 선택했는지 확인.  
     → 해당 서비스 **Environment Variables** 에 `CRON_SECRET`, `API_BASE_URL` 이 반드시 들어 있어야 합니다.  
     → Coolify가 크론 실행 시 컨테이너에 환경 변수를 주입하지 않을 수 있으므로, **서비스 환경 변수**에 두 값을 넣고 재배포한 뒤 다시 실행.
   - **Run on server (SSH)** 인 경우: SSH로 실행되는 셸에는 **크론용 환경 변수가 없을 수 있음**.  
     → Command 에 값을 직접 넣지 말고, **스크립트를 쓰는 경우** 실행 전에 `export API_BASE_URL=https://api.nanum.online` 등으로 한 줄 넣거나,  
     → **한 줄 curl** 로 테스트할 때는 `API_BASE_URL` 과 `CRON_SECRET` 을 **Command 필드에 직접** 넣어 보세요.  
       예: `curl -s -X POST -H "X-Cron-Secret: 여기에_실제_시크릿" -H "Content-Type: application/json" "https://api.nanum.online/api/v1/admin/jobs/cron/verifying-timeout"`

2. **환경 변수 주입 여부 확인**
   - Coolify Cron Job / Scheduled Task 설정에 **Environment** 또는 **Environment Variables** 항목이 있으면  
     `CRON_SECRET`, `API_BASE_URL` 을 거기서 설정하고, Command 에서는 `$CRON_SECRET`, `$API_BASE_URL` 만 사용.
   - **컨테이너 이름**이 실제 GEMS_OCR 앱 컨테이너와 **완전히** 일치하는지 확인. "GEMS"처럼 일부만 넣으면 안 됨. Coolify → 해당 서비스 → 컨테이너/인스턴스 목록에 나오는 이름(예: `gems-ocr`, `프로젝트명-서비스명-1`)을 그대로 복사해 사용 (다른 컨테이너면 env가 없어 exit 3 발생).

3. **한 줄 curl로 테스트**
   - Command 를 아래처럼 **고정 URL·고정 시크릿**으로 바꿔서 한 번 실행해 보기.  
     (실제 값으로 치환)
     ```bash
     curl -s -o /dev/null -w "%{http_code}" -X POST -H "X-Cron-Secret: 실제CRON_SECRET값" -H "Content-Type: application/json" "https://api.nanum.online/api/v1/admin/jobs/cron/verifying-timeout"
     ```
     응답이 `200` 이면 API·네트워크는 정상. 이때 다시 `$API_BASE_URL` / `$CRON_SECRET` 을 쓰도록 바꾸고, **크론 쪽에 환경 변수가 들어가도록** Coolify 설정을 조정.

4. **스크립트 사용 시**
   - `cron_verifying_timeout.sh` 를 쓰면, **실행되는 셸에** `CRON_SECRET` 과 `API_BASE_URL` 이 export 되어 있어야 합니다.  
     Coolify SSH 실행 시 홈 디렉터리 `.profile` / `.bashrc` 가 로드되지 않을 수 있으므로, 스크립트 **맨 위**에 다음 두 줄을 넣어 보세요 (값은 실제 값으로).
     ```bash
     export API_BASE_URL="https://api.nanum.online"
     export CRON_SECRET="실제_시크릿_값"
     ```

**정리:** exit code 3 = URL 잘못됨 → **크론이 도는 환경에 `API_BASE_URL`(과 필요 시 `CRON_SECRET`)이 확실히 들어가 있는지** 확인하면 해결됩니다.
