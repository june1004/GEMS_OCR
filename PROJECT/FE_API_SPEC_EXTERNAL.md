# GEMS 영수증 인증 연동 요약 (외부 공유용)

> 프론트엔드·외부 파트너가 **어떤 순서로, 어떤 정보를 보내면 되는지** 요약한 문서입니다.  
> 상세 기술 스펙은 `FE_API_SPEC_ONE_PAGE.md`를 참고해 주세요.

---

## 1. 전체 흐름 (4단계)

| 단계 | 담당 | 하는 일 |
|------|------|--------|
| 1 | 화면(클라이언트) | **업로드 주소 발급 요청** — 파일명, 사용자 식별자, **유형(숙박/관광)** 전달 후, 서버에서 받은 **신청 번호(receiptId)**와 **이미지별 업로드 주소** 수신 |
| 2 | 화면(클라이언트) | **이미지 업로드** — 받은 주소로 각 이미지 파일 전송 |
| 3 | 화면(클라이언트) | **검증 요청** — **신청 번호**와 **올린 이미지 목록(경로 + 종류)**을 한 번에 서버로 전송 |
| 4 | 화면(클라이언트) | **결과 확인** — 같은 **신청 번호**로 서버에 결과 조회 (적합/부적합, 금액, 사유 등) |

**요약:** 이미지는 **한 장씩** 올리되, **한 건의 신청**으로 묶어서 제출하고, **하나의 신청 번호**로 결과를 받습니다.

---

## 2. 유형별 구분 (반드시 준수)

| 유형 | 서버에 보낼 이름 | 의미 | 제출 가능 이미지 |
|------|------------------|------|------------------|
| 숙박 | **STAY** | 숙박 비용 증빙 | 영수증 1장 필수, OTA 명세서 1장 선택 |
| 관광 소비 | **TOUR** | 관광 소비 증빙 | 영수증 1~3장 (합산) |

- 업로드 주소 발급·검증 요청 시 **유형**은 반드시 **STAY** 또는 **TOUR** 중 하나로 전달합니다.  
- **SPEND** 등 다른 값은 사용하지 않습니다.

---

## 3. 이미지 개수와 신청 번호(receiptId)

- **이미지 1장** → 업로드 주소 1번 발급 → **신청 번호 1개** 발급 → 검증 요청 1번 → **결과 1건**.
- **이미지 여러 장(한 건으로 묶을 때)**  
  - **첫 이미지**에서 받은 **신청 번호(receiptId)**를 저장하고,  
  - **두 번째·세 번째 이미지** 업로드 주소 발급 시 **같은 신청 번호**를 함께 보내면,  
  - **한 건의 신청**으로 묶이고, 검증 요청 1번 → **결과 1건**만 받습니다.

즉, **이미지마다 신청 번호를 새로 만들면 결과도 건별로 따로** 나오고, **같은 신청 번호로 묶으면 결과는 1건**으로 나옵니다.

---

## 4. 검증 요청 시 보내는 정보 (개념)

- **신청 번호(receiptId)**  
- **사용자 식별자(userUuid)**  
- **캠페인 번호(campaignId)** — 없으면 1  
- **유형(type)** — STAY 또는 TOUR  
- **이미지 목록(documents)**  
  - 각 항목: **저장된 이미지 경로(imageKey)** + **종류(docType)**  
  - 종류: **RECEIPT**(일반 영수증), **OTA_INVOICE**(숙박 OTA 명세서)  
- STAY: RECEIPT 1장 필수, OTA_INVOICE는 있을 때만 추가  
- TOUR: RECEIPT만 1~3장

---

## 5. 결과 조회 시 받는 정보 (개념)

- **판정(status)** — 적합(FIT) / 부적합(UNFIT) / 중복(DUPLICATE) / 오류(ERROR)  
- **금액(amount)**  
- **부적합 사유(failReason)** — 부적합일 때만  
- **리워드 금액(rewardAmount)**  
- **가맹점 주소(address)**  
- **카드 앞 4자리(cardPrefix)** (비식별)

---

## 6. 필수 확인 사항 (연동 전)

- [ ] 유형은 **STAY**, **TOUR**만 사용하는가?  
- [ ] 여러 장을 한 건으로 묶을 때, **첫 발급에서 받은 신청 번호**를 저장해 두고, 추가 이미지 발급·검증 요청에 **같은 번호**를 쓰는가?  
- [ ] 검증 요청 시 **이미지 경로 + 종류(RECEIPT/OTA_INVOICE)**를 빠짐없이 넘기는가?  
- [ ] 검증 요청 후 **같은 신청 번호**로 결과 조회를 하고 있는가?
