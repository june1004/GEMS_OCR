## 3. API·코드 반영 요약

- **FE 연동**: Presigned → 업로드(PUT) → Complete(documents, optional data.items[]) → Status(폴링/콜백).
- **Complete data**: `data.items[]` 지원. 있으면 `user_input_snapshot` 저장, 관리자 상세에 노출.
- **관리자**: 판정 규칙(유효일 포함), 상세(user_input_snapshot), override, 콜백 재전송, 이미지 presigned — `FE_API_규격_문서.md`·`관리자_페이지_추가_개발_지침.md` 참고.
- **마이그레이션**: `submissions_user_input_snapshot.sql`, `judgment_rule_config_minio_days.sql` 등 `migrations/` 참고.

## 4. 정리 원칙

- **FE 개발사 전달**: `FE_API_규격_문서_외부전달용.md` + (선택) `테스트용_FE_적용_가이드.md`. 관리자 API·내부 정책 문서는 전달하지 않음.

- **관리자 개발**: `ADMIN_PORTAL_GUIDE.md` + **`관리자_페이지_추가_개발_지침.md`** 를 추가 개발 지침으로 사용.
