# 크론 잡 설정 (Coolify 등)

## 1. curl exit code 3 의 의미

**exit code 3** 은 curl에서 **"URL malformed"(URL 형식 오류)** 일 때 나옵니다.

즉, `$API_BASE_URL` 이 비어 있거나 잘못되어 있어서  
`https:///api/v1/...` 같은 잘못된 URL로 요청이 나가는 경우가 많습니다.

---

## 2. Coolify에서 확인할 것

크론 잡이 실행되는 **그 잡의 환경 변수**에 아래가 **반드시** 들어 있는지 확인하세요.

| 변수 | 설명 |
|------|------|
| **API_BASE_URL** | 예: `https://your-backend.nanum.online` — **끝에 `/` 없이**, 프로토콜 포함 **전체 URL** 만 넣습니다. |
| **CRON_SECRET** | 나중에 백엔드 `.env` 에 넣을 값과 **동일한 값**으로 설정합니다. |

- Coolify에서는 **"Cron Job" 설정 화면의 Environment** 에 위 변수들을 넣어야 합니다.
- **일반 앱의 환경 변수와는 별도로**, **크론 잡 전용** 환경 변수를 설정해야 합니다.

---

## 3. 백엔드에 크론 API 반영 (GEMS_OCR)

이 프로젝트에는 아래 엔드포인트와 **CRON_SECRET** 검증이 이미 포함되어 있습니다.

| 항목 | 내용 |
|------|------|
| **라우트** | `main.py` — `POST /api/v1/admin/jobs/cron/verifying-timeout` |
| **인증** | `X-Cron-Secret` 헤더가 서버의 **CRON_SECRET** 과 일치할 때만 동작 |
| **미설정 시** | 서버에 `CRON_SECRET` 이 없으면 **503** (CRON_SECRET not configured) 반환 |

서버 **.env** 에 다음을 넣어야 크론이 정상 동작합니다.

```env
CRON_SECRET=여기에_강한_비밀값_설정
```

Coolify 크론의 **CRON_SECRET** 과 **완전히 같은 값**이어야 합니다.

### 동작 (GEMS_OCR 기준)

- 판정 규칙(`judgment_rule_config`)의 **verifying_timeout_minutes** 가 0보다 클 때만 동작 (0이면 비활성).
- **VERIFYING** / **PENDING_VERIFICATION** 상태로 해당 분 수 초과한 제출을 찾아
- **verifying_timeout_action** 에 따라 `status` 를 **UNFIT** 또는 **ERROR** 로 변경하고, FE 콜백 URL 로 전송합니다.

---

## 4. 크론 명령 예시 (올바른 형태)

```bash
curl -s -X POST \
  -H "X-Cron-Secret: $CRON_SECRET" \
  -H "Content-Type: application/json" \
  "$API_BASE_URL/api/v1/admin/jobs/cron/verifying-timeout"
```

- Coolify에서 위 명령을 사용할 때, **같은 Cron Job 의 Environment** 에 `API_BASE_URL`, `CRON_SECRET` 을 설정하세요.

---

## 5. 환경 변수 요약

| 변수 | 위치 | 설명 |
|------|------|------|
| **CRON_SECRET** | 백엔드 서버 (.env) | 크론 요청 검증용. Coolify 크론과 **동일한 값** 사용. |
| **API_BASE_URL** | Coolify **Cron Job** Environment | 백엔드 URL (예: `https://api.nanum.online`), 끝에 `/` 없이. |
| **CRON_SECRET** | Coolify **Cron Job** Environment | 백엔드와 동일한 시크릿 (헤더에 전달). |
